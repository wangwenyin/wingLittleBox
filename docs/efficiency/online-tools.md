# 在线工具

在线工具最重要的标准，不是“功能多”，而是打开快、结果准、广告少、能真正进入日常工作流。很多人收藏了一堆链接，真正常用的其实只有很少几类。

这篇更关注“什么类型值得常驻”，而不是堆砌网址。

## 什么时候在线工具最有价值

在线工具最适合解决这类问题：

- 偶发但反复出现的小任务
- 不值得为此单独安装本地软件
- 需要跨设备即开即用

如果一个能力你每天重度使用，很多时候本地工具反而更稳；如果只是高频碎任务，在线工具就很合适。

## 开发类

### JSON 工具

这是开发者最常用的一类在线工具之一。

高频能力包括：

- JSON 格式化
- JSON 转 TypeScript
- JSON Path 测试

这类工具值钱的地方不在“能不能做”，而在于：

- 打开即用
- 输入输出清晰
- 能快速辅助接口联调和结构整理

### 编解码工具

很适合放进常用收藏夹的包括：

- Base64
- URL Encode / Decode
- 时间戳转换

这些任务都很碎，但出现频率很高。把它们集中到一两个可靠入口里，比每次临时搜索要省事得多。

### 正则调试

正则工具最有用的地方，是让你快速看见匹配结果，而不是在本地一遍遍试。

重点能力通常包括：

- 正则测试
- 高亮匹配结果
- 分组查看

## 设计与协作类

这类工具很适合处理“临时需要，但不值得为此打开重型软件”的任务：

- 颜色选择与对比度检查
- 图片压缩
- SVG 在线编辑
- 流程图工具

如果你经常写文档、做简单视觉标注或快速画流程，这些工具会很实用。

## 文本处理类

对技术写作和开发协作来说，文本类在线工具的价值往往被低估。

常见高频项包括：

- Markdown 预览
- 文本 diff 对比
- 代码片段分享

它们很适合：

- 快速检查文档渲染
- 对比配置差异
- 临时共享片段给同事

## 选择与收藏原则

不要把在线工具当成“看见一个就收藏一个”的清单。更实用的方式是：

- 每类只保留 1 到 2 个主力入口
- 收藏按用途分组，而不是按网站名分组
- 定期清理失效、广告变重或体验变差的链接

## 使用建议

- 常用工具做统一收藏
- 涉及隐私数据时避免直接上传第三方网页
- 能本地化运行的优先本地化
- 高频任务优先选最快打开、交互最少的工具

## 小结

在线工具真正的价值，不是“功能百科全书”，而是成为你工作流里的快速补位工具。只要你把 JSON、编码、正则、Markdown、diff 这几类入口稳定下来，日常碎任务的处理速度就会快很多。
