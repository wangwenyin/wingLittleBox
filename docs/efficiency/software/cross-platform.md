# 多平台软件

多平台软件最大的价值，不是“我在哪台机器上都能装”，而是它能帮助你把知识、文件、开发环境和工作流带着走。只要跨设备切换频繁，这类工具通常比平台专属小工具更值得优先打磨。

这篇重点不是列全，而是挑那些更容易长期保留、跨系统迁移成本低的类型。

## 先看选择标准

多平台工具要不要留下来，我通常优先看这几件事：

- 同步和迁移能力是否稳定
- 数据是否掌握在自己手里
- 是否能进入高频工作流
- 换设备后恢复成本是否低

真正有价值的多平台工具，应该能减少“环境重建”的摩擦。

## 笔记与知识管理

### Obsidian

如果你更看重本地可控、长期沉淀和 Markdown 生态，Obsidian 很值得保留。

它的优势在于：

- 基于本地 Markdown 文件
- 双向链接和知识图谱能力强
- 迁移成本低
- 很适合做个人长期知识库

它更适合：

- 写技术笔记
- 建个人 wiki
- 不希望知识被平台强绑定

### Notion

Notion 更像是偏在线化、偏协作型的信息组织工具。

它的优势在于：

- 页面组织能力强
- 数据库视图灵活
- 文档协作和项目管理体验好

如果你的重点是团队协作、项目管理、内容共享，它会更顺；如果你的重点是长期本地知识沉淀，Obsidian 往往更稳。

### 如何取舍

- 个人知识库优先本地可控：更偏 Obsidian
- 协作和项目管理优先：更偏 Notion

很多人最后会两者并存，但职责最好分开，不要把同一类内容重复堆两套系统。

## 开发工具

### Visual Studio Code

VS Code 之所以是典型多平台主力，不只是因为能装在多台机器上，而是因为它的配置、插件和项目工作流相对容易迁移。

它适合：

- 多语言开发
- 多项目切换
- 轻量到中等复杂度工程

如果你频繁在不同设备间切换，编辑器一致性能显著降低适应成本。

### Git

Git 本身不是“效率软件”那种直观工具，但它几乎是跨设备工作流最重要的基础设施之一。

它的价值在于：

- 代码与文档版本可追踪
- 不同设备间状态同步清晰
- 回滚、对比、协作都有统一机制

真正高效的不是“会几个命令”，而是把版本管理变成日常默认动作。

## 文件同步

### Syncthing

Syncthing 很适合那些更在意私有化、控制权和多设备直连同步的人。

它的优点是：

- 点对点同步
- 更偏私有化
- 对本地文件体系友好

如果你已经习惯本地目录驱动工作流，它会很自然。

### 坚果云 / Dropbox / OneDrive

这类云同步服务更适合：

- 文档同步
- 跨设备访问
- 需要版本历史和共享能力

选择时重点看：

- 版本历史是否可靠
- 冲突处理策略是否清晰
- 跨平台客户端是否稳定
- 价格和容量是否匹配你的使用方式

## 更实用的搭配思路

如果你想减少跨设备切换的心智负担，可以优先把下面几类东西做成“平台无感”：

- 知识库
- 编辑器配置
- 代码仓库
- 常用文档目录
- 基础命令与脚本

这样无论是 Windows 还是 Mac，进入工作状态都会快很多。

## 小结

多平台软件最核心的价值，是让你的工作流不被设备绑住。对大多数人来说，优先打磨下面这几块已经足够有收益：

- 一套长期可维护的笔记系统
- 一套可迁移的编辑器环境
- 一套稳定的文件同步方案

剩下的工具选择，才是在这套骨架上的局部优化。
