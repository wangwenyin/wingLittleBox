# 浏览器设置与扩展

浏览器是开发者最常驻的工作界面之一。你查资料、调接口、看文档、开后台、调试页面，绝大部分时间都离不开它。很多效率问题看起来像“工作节奏问题”，其实只是浏览器环境过于混乱。

这篇的重点是建立一套稳定、克制、长期可维护的浏览器工作区。

## 先做基础设置，而不是先装扩展

真正影响体验的第一步，往往不是装十几个插件，而是先把浏览器本身设置好。

优先建议：

- 固定常用标签页
- 开启关闭标签页恢复能力
- 单独管理工作与个人 Profile
- 接入密码管理能力
- 对下载目录和默认打开行为做清晰约定

如果工作和个人环境混在同一个 Profile 里，书签、登录态、历史记录和扩展权限都会越来越乱。

## 用 Profile 隔离上下文

如果你同时处理公司项目、个人写作、临时测试和日常浏览，强烈建议至少拆成两个 Profile：

- 工作
- 个人

更进一步还可以拆：

- 工作主环境
- 测试环境
- 个人环境

这样做的价值很直接：

- 登录态互不干扰
- 书签和历史更干净
- 插件权限更容易控制
- 调试问题时上下文更稳定

## 推荐扩展方向

插件不求多，求明确分工。下面这些方向更值得优先考虑。

### 广告与干扰屏蔽

#### `uBlock Origin`

这是非常值得长期保留的一类扩展。

它的价值不只是“去广告”，还包括：

- 减少页面噪音
- 提升阅读专注度
- 降低一些网页的资源浪费

对信息检索密集型用户来说，体验提升很明显。

### Markdown / 技术内容阅读

这类扩展的价值在于让原本难读的页面结构更清爽，例如：

- 沉浸式阅读类扩展
- JSON 格式化查看类扩展

对于看接口返回、看技术文档、看长文教程，这类能力几乎每天都会用到。

### 开发辅助

开发辅助插件应该只保留最有价值的那几类：

- Vue Devtools
- React DevTools
- 请求抓包与接口调试辅助扩展

原则是：只装你当前技术栈真正需要的。

如果你主要写 Vue，就别同时堆一堆不相关框架插件，避免权限膨胀和性能噪音。

### 标签管理

很多时候不是你开的页面真的有用，而是你不敢关。

这类工具适合解决“先收起来，稍后再处理”的问题：

- OneTab 一类的标签收纳工具

它不是必须品，但如果你经常在几十个标签页之间来回切，会很有帮助。

## 开发者视角下更重要的几个细节

### 把 DevTools 用熟

很多问题其实根本不需要额外扩展，浏览器开发者工具已经够强：

- 元素检查
- 网络请求
- 控制台
- 性能录制
- 存储查看

扩展只能增强局部场景，DevTools 才是基础设施。

### 少装“看起来很强”的全能扩展

这类扩展的问题通常是：

- 权限范围大
- 资源占用不透明
- 一段时间后你根本忘了它为什么装

浏览器环境越核心，越要克制。

## 使用原则

- 扩展不求多，求稳定
- 同类扩展只保留一套主方案
- 定期清理不再使用的扩展
- 对隐私敏感扩展单独做权限审查
- 工作环境优先追求确定性，不要频繁试新

## 小结

浏览器效率提升的关键，不在于装更多扩展，而在于：

- 用 Profile 隔离上下文
- 用最少的一组扩展覆盖最常见场景
- 把 DevTools 当成默认调试入口

只要环境干净，日常检索、阅读和调试的切换成本会低很多。
