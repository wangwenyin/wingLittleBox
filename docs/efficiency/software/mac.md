# Mac 平台

Mac 上的效率提升，通常不是靠某一个“神器”完成的，而是靠一组稳定、互相配合的工具链：启动器、窗口管理、终端、剪贴板、文件同步和自动化。只要这套链路顺了，日常操作的中断会明显减少。

这篇的重点不是列尽所有工具，而是整理一套更容易长期保留的选择思路。

## 先搭工作流骨架

如果是新机器，我更建议按这个顺序搭环境：

1. 启动器
2. 窗口管理
3. 终端与开发环境
4. 剪贴板与输入增强
5. 自动化工具

原因很简单：这些环节几乎覆盖了你每天最频繁的操作，收益比“装几十个小工具”更直接。

## 启动与搜索

### Raycast

对多数开发者来说，Raycast 已经足够作为默认入口。

适合它的原因：

- 启动应用速度快
- 搜索文件、命令、设置都比较顺手
- 插件生态丰富
- 可以把剪贴板、窗口、脚本和系统控制统一到一个入口

如果你希望减少“点图标”和“翻文件夹”的动作，它通常是最先值得装的工具之一。

### Alfred

Alfred 更像是另一种风格的老牌启动器。

它的优势在于：

- 工作流能力成熟
- 社区沉淀时间长
- 高度自定义空间大

如果你愿意花时间折腾自己的命令入口，Alfred 依然很强；如果更想开箱即用，Raycast 往往更省心。

## 窗口管理

### Rectangle

Rectangle 的价值在于轻量、直接、几乎没有学习成本。

适合场景：

- 外接显示器办公
- 需要频繁左右分屏
- 想把窗口管理彻底键盘化

### Magnet

Magnet 的体验也比较稳定，更适合：

- 更偏图形界面的使用习惯
- 不想折腾太多高级能力
- 只想快速获得可靠分屏

### 选择建议

- 想轻量免费，优先 Rectangle
- 想简单稳定的商业方案，可以考虑 Magnet

## 终端与开发

### iTerm2

iTerm2 依旧是 Mac 终端生态里最稳的一类选择。

它的价值不在于“能跑命令”，而在于这些细节：

- 标签页和分屏成熟
- Profiles 方便区分项目环境
- 快捷键和外观定制空间大
- 与 `zsh`、`tmux`、SSH 工作流结合顺畅

### OrbStack / Docker Desktop

如果你有容器开发需求，这一组工具几乎绕不开。

选择时重点看：

- 资源占用
- 文件系统性能
- 与现有项目的兼容性
- 团队内部是否已有统一方案

对个人开发体验来说，很多人会更在意更轻的占用和更快的响应。

## 补充建议

虽然这页还没单独拆剪贴板和自动化工具，但工作流上我建议优先补这两类能力：

- 剪贴板历史
- 文本片段管理
- 快速脚本执行
- 文件和窗口快捷操作

因为它们会直接减少重复性微操作。

## 一套更实用的使用原则

- 优先建立快捷键体系，而不是依赖鼠标路径
- 常驻工具不求多，求高频且稳定
- 同类工具只保留一套主方案，避免功能重叠
- 把启动器、窗口管理、终端、剪贴板串成一个连续流程

## 小结

Mac 平台的效率提升，本质上是把“打开、切换、搜索、执行”这几个动作缩短。只要你先把启动器、窗口管理和终端体系搭好，后面的体验会稳定很多。
