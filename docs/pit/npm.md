# npm 踩坑记录

npm 相关问题通常不是“不会装包”，而是版本、锁文件、镜像源、权限和依赖树不一致导致的。

## 常见问题

### 依赖安装后仍然报找不到模块

优先检查：

- 是否装在当前工作目录
- 是否装成了 `devDependencies`
- 锁文件和 `node_modules` 是否已失配
- Monorepo 下是否装到了错误包

### 安装速度慢或失败

可排查：

- 镜像源配置
- 网络代理
- 是否命中私有源权限问题
- 是否存在可选依赖平台兼容问题

### 版本冲突

遇到 peer dependency 冲突时，不要只看最上层报错，要看：

- 当前包需要什么版本
- 你的工程里实际装了什么版本
- 是直接依赖冲突，还是间接依赖冲突

## 建议

- 团队统一 Node 与包管理器版本
- 提交锁文件
- 不混用多个包管理器
- 安装前先确认依赖应放在生产还是开发依赖
