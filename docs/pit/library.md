# 第三方库踩坑记录

第三方库的问题，很多都不是“库本身坏了”，而是接入边界、版本不匹配或默认行为与业务预期不一致。

## 常见排查方向

- 文档版本和安装版本是否一致
- 是否区分默认导出与命名导出
- 是否存在 SSR 兼容问题
- 是否依赖浏览器环境对象
- 是否需要样式文件或插件初始化

## 接入建议

- 首次接入先做最小 Demo
- 不要直接把库能力铺满到业务全局
- 对关键行为做二次封装，减少未来替换成本
- 记录你真正用到的 API，而不是把整份文档搬进项目

## 一个常见误区

看到报错就直接换库，通常太早。先确认：

- 是使用方式错了
- 还是库的能力边界不适合当前场景
